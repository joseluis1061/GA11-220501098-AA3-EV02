Buenas Prácticas de Calidad Aplicadas

Basadas en Scrum, DevOps y el estándar ISO 25010:

  • Criterios de aceptación claros por funcionalidad.
  • Integración continua para validar calidad constante.
  • Revisión por pares (code review).
  • Pruebas automatizadas para asegurar fiabilidad.
  • Documentación clara de métricas y hallazgos.

Características, Subcaracterísticas y Métricas de Calidad

Evaluación basada en ISO/IEC 25010:

  • Funcionalidad: Completitud, precisión.
    Métricas: cobertura de requisitos, errores funcionales.
  • Fiabilidad: Estabilidad operacional.
    Métricas: fallos por módulo, tiempo promedio sin fallos.
  • Eficiencia: Tiempos y recursos.
    Métricas: tiempo de respuesta, uso de memoria.
  • Usabilidad: Facilidad de aprendizaje y uso.
    Métricas: pasos por tarea, accesibilidad.
  • Mantenibilidad: Facilidad de corrección y evolución.
    Métricas: complejidad ciclomática, modularidad.

Herramientas Software para Automatización de Medición y Evaluación

Para garantizar una evaluación precisa y automatizada del Sistema de Inventario, se utilizaron las siguientes herramientas:

  • SonarQube: Análisis estático de código, métricas de mantenibilidad, complejidad ciclomática y vulnerabilidades.
  • JUnit / Jest: Automatización de pruebas unitarias para validar fiabilidad y comportamiento funcional.
  • Postman: Automatización de pruebas de API, validación de respuestas y tiempos de ejecución.
  • Lighthouse: Medición de rendimiento, accesibilidad y buenas prácticas del frontend.
  • GitHub Actions: Automatización del flujo CI/CD para aplicar reglas de calidad y ejecutar pruebas en cada commit.
  • Excel / Google Sheets: Visualización de resultados mediante tablas, gráficos y consolidación de métricas.

Estas herramientas permitieron automatizar mediciones clave, aplicar criterios de evaluación definidos y visualizar resultados de manera clara y cuantificable.

Actividades y Tareas del Proceso de Evaluación

Se realizaron las siguientes actividades:

  • Análisis del código y estructura del proyecto.
  • Ejecución de pruebas unitarias, integración y sistema.
  • Automatización de métricas mediante SonarQube y otras herramientas.
  • Evaluación manual y automatizada de usabilidad.
  • Comparación de resultados con estándares esperados.
  • Documentación final de hallazgos y conclusiones.

Conclusiones

La evaluación de calidad del Sistema de Inventario confirmó que cumple con los requisitos principales de funcionalidad, fiabilidad y mantenibilidad. Gracias al uso de herramientas automatizadas, los resultados fueron medibles, precisos y reproducibles.
